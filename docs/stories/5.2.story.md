# Story 5.2 — Auto-save no Supabase (snapshots periódicos)

**Epic:** EP-05 — Persistência & Salvamento
**Story ID:** EP05-S02
**Status:** Draft
**Points:** 3
**Branch:** `story/5.2-auto-save`

## Contexto

O nível individual já é salvo por `useLevelUpdate` a cada tap (Story 4.2). Esta story adiciona snapshots periódicos da partida inteira para a tabela `level_snapshots` — um backup histórico completo a cada 15 minutos. Também atualiza `lastSavedAt` no store após cada snapshot bem-sucedido.

## Notas Técnicas

**`captureSnapshot`:**
```typescript
export async function captureSnapshot(
  supabase: SupabaseClient<Database>,
  gameId: string,
  players: GamePlayerWithInfo[],
): Promise<void> {
  const snapshots = players.map(p => ({
    game_id: gameId,
    player_id: p.player_id,
    level: p.level,
  }));
  const { error } = await supabase.from('level_snapshots').insert(snapshots);
  if (error) throw new Error(`Failed to capture snapshot: ${error.message}`);
}
```

**`useAutoSave`:**
```typescript
export function useAutoSave() {
  const { activeGame, gamePlayers, setLastSavedAt } = useGameStore();
  useEffect(() => {
    if (!activeGame) return;
    const interval = setInterval(async () => {
      try {
        await captureSnapshot(supabase, activeGame.id, gamePlayers);
        setLastSavedAt(new Date().toISOString());
      } catch (e) {
        console.error('Auto-save failed:', e);
      }
    }, GAME_CONFIG.SNAPSHOT_INTERVAL_MS);
    return () => clearInterval(interval);
  }, [activeGame?.id]);
}
```
