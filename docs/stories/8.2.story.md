# Story 8.2 — Drag & Drop com @dnd-kit

**Epic:** EP-08 — Ordenação & Drag-and-Drop
**Story ID:** EP08-S02
**Status:** InReview
**Points:** 5
**Branch:** `story/8.2-drag-drop`

## Contexto

Quando `sortMode === 'custom'`, o `PlayerGrid` habilita drag-and-drop via `@dnd-kit`. Arrastar um card reordena otimisticamente no store; ao soltar, a nova ordem é persistida no Supabase (`games.player_order`).

## Acceptance Criteria

- [x] AC1: `PlayerGrid` usa `DndContext` + `SortableContext` quando `sortMode === 'custom'`; em outros modos, renderiza normalmente sem DnD
- [x] AC2: Arrastar um card reordena os cards visualmente durante o drag
- [x] AC3: Após soltar (`onDragEnd`), `gamePlayers[].position` é atualizado no store na nova ordem
- [x] AC4: Após soltar, `updateGameOrder(supabase, gameId, playerOrder)` persiste `player_order` no Supabase
- [x] AC5: `DragOverlay` exibe card fantasma com `opacity-70` durante o drag
- [x] AC6: `npm run lint` e `npm run typecheck` passam sem erros

## Tasks de Implementação

- [x] T1: Adicionar `updateGameOrder(supabase, gameId, playerOrder: string[])` em `packages/shared/src/services/games.ts` e exportar via `index.ts`
- [x] T2: Atualizar `PlayerGrid` com `DndContext`, `SortableContext` e `DragOverlay`; usar `useSortable` em cada card wrapper
- [x] T3: Adicionar `setGamePlayersOrder(orderedIds: string[])` action no Zustand store
- [x] T4: Executar `npm run lint` e `npm run typecheck`

## Notas Técnicas

PointerSensor com `activationConstraint: { distance: 8 }` evita drag acidental ao clicar nos botões. `rectSortingStrategy` para layout em grid.

## Dependências

Story 8.1 (SortDropdown / sortMode). Zustand store (EP-05).
