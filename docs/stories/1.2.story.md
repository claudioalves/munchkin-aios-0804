# Story 1.2 — Setup Supabase project

**Epic:** EP-01 — Setup & Infraestrutura
**Story ID:** EP01-S02
**Status:** Draft
**Points:** 2
**Branch:** `story/1.2-supabase-setup`

## Contexto

Antes de qualquer migration ou funcionalidade de auth poder ser implementada, o projeto Supabase precisa existir e estar conectado ao ambiente de desenvolvimento local. Esta story cobre a criação manual do projeto no Supabase Dashboard, a configuração das variáveis de ambiente no app web e a geração dos tipos TypeScript do banco — o contrato tipado entre o frontend e o banco de dados.

Os tipos gerados em `packages/shared/src/database.types.ts` são o alicerce da type safety end-to-end do projeto. Toda chamada ao Supabase client (`supabase.from('players')`) usará esses tipos para garantir que o compilador capture erros de schema antes do runtime.

## Notas Técnicas

**Localização do `project-ref`:** No Supabase Dashboard → Settings → General → Reference ID. Formato: string alfanumérica curta como `abcdefghijklmnop`.

**Comando de geração de tipos:**
```bash
supabase gen types typescript --linked > packages/shared/src/database.types.ts
```
O arquivo gerado deve ser re-executado sempre que o schema do banco mudar (após cada migration).

**Export no `packages/shared/src/index.ts`:**
```typescript
export type { Database } from './database.types';
```

**Atenção com tipos gerados:** O arquivo `database.types.ts` é auto-gerado — nunca editá-lo manualmente. Se o banco ainda estiver vazio (sem tabelas), o arquivo gerado terá um schema vazio `{}` — isso é esperado nesta story. As tabelas reais serão criadas na Story 1.3.

**Variáveis de ambiente no Vite:** Apenas variáveis prefixadas com `VITE_` são expostas ao browser. Nunca usar chaves secretas (service role key) — apenas a `anon key`.

**Referências:** `docs/framework/tech-stack.md` (seção 3 — Variáveis de Ambiente), Supabase CLI docs: `https://supabase.com/docs/reference/cli`
