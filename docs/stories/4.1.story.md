# Story 4.1 — PlayerCard component

**Epic:** EP-04 — Tela de Partida — Core
**Story ID:** EP04-S01
**Status:** Draft
**Points:** 5
**Branch:** `story/4.1-player-card`

## Contexto

O PlayerCard é o componente central do app — o "protagonista visual" da tela de partida. O número de nível ocupa 60% da altura do card, em font-display. O card muda visualmente quando o jogador é líder (borda dourada pulsante) ou vencedor (borda esmeralda + ícone de troféu). Esta story implementa o PlayerCard completo com os botões +/- integrados.

## Acceptance Criteria

- [ ] AC1: `PlayerCard` renderiza: avatar circular com inicial+cor, nome (truncado), número de nível em font-display grande, botões − e +
- [ ] AC2: Botão + desabilitado quando `level >= maxLevel`; botão − desabilitado quando `level <= 1`
- [ ] AC3: `isLeader=true` aplica borda `ring-2 ring-brand-gold` e animação `animate-gold-pulse`
- [ ] AC4: `isVictory=true` aplica borda `ring-2 ring-brand-emerald`, exibe badge "🏆 Vitória!" e desabilita ambos os botões
- [ ] AC5: Botões têm tap target mínimo 64px (h-16 = 64px)
- [ ] AC6: Toque no botão dispara feedback visual imediato (`active:scale-90`)
- [ ] AC7: `npm run lint` e `npm run typecheck` passam sem erros
