# Story 5.4 — Restauração de estado ao abrir app

**Epic:** EP-05 — Persistência & Salvamento
**Story ID:** EP05-S04
**Status:** Draft
**Points:** 3
**Branch:** `story/5.4-state-restore`

## Contexto

Ao abrir o app, o Zustand store hidrata do localStorage automaticamente (persist middleware). Mas podem existir escritas mais recentes no Supabase (ex: o app travou antes de salvar no localStorage). Esta story implementa uma reconciliação de startup: compara o `lastSavedAt` do localStorage com o `updated_at` do jogo no Supabase, e usa a fonte mais recente.

## Acceptance Criteria

- [ ] AC1: Hook `useGameRestore` existe em `apps/web/src/hooks/useGameRestore.ts`
- [ ] AC2: Na inicialização, busca `getActiveGame()` do Supabase
- [ ] AC3: Se o Supabase retornar jogo mais recente que o localStorage (`updated_at` > `lastSavedAt`), atualiza o store
- [ ] AC4: Se localStorage tiver dados mais recentes ou iguais, mantém localStorage (não sobrescreve)
- [ ] AC5: Se Supabase retornar `null` mas store tiver `activeGame`, considera que o jogo foi finalizado em outro dispositivo — chama `clearGame()`
- [ ] AC6: `useGameRestore` chamado em `main.tsx` após a auth anônima
- [ ] AC7: `npm run lint` e `npm run typecheck` passam sem erros

## Notas Técnicas

**Lógica de restauração em `main.tsx`:**
```typescript
// Após ensureAnonymousSession:
const { activeGame, lastSavedAt, setActiveGame, setGamePlayers, clearGame } = useGameStore.getState();
const supabaseGame = await getActiveGame(supabase).catch(() => null);

if (!supabaseGame && activeGame) {
  // Jogo foi encerrado em outro dispositivo
  clearGame();
} else if (supabaseGame) {
  const supabaseUpdatedAt = supabaseGame.game_players[0]?.updated_at ?? supabaseGame.started_at;
  const localSavedAt = lastSavedAt ?? '1970-01-01';
  if (supabaseUpdatedAt > localSavedAt) {
    // Supabase é mais recente
    setActiveGame(supabaseGame);
    setGamePlayers(supabaseGame.game_players);
  }
  // Caso contrário: localStorage é mais recente, mantém
}
```
