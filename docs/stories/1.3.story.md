# Story 1.3 — Migração inicial do banco

**Epic:** EP-01 — Setup & Infraestrutura
**Story ID:** EP01-S03
**Status:** Draft
**Points:** 3
**Branch:** `story/1.3-initial-migration`

## Contexto

Esta story cria o schema completo do banco de dados que suporta toda a lógica do Munchkin Level Tracker. O modelo relacional cobre as quatro entidades centrais: `players` (jogadores cadastrados), `games` (partidas), `game_players` (relação entre jogadores e partidas, com o nível atual) e `level_snapshots` (histórico de níveis para o gráfico de progresso).

Além das tabelas, esta story configura Row Level Security (RLS) em todas as tabelas — garantindo que cada usuário anônimo acesse apenas seus próprios dados — e cria o trigger `check_level_limit` que impede níveis fora do intervalo válido do jogo (1-20 no modo normal, 1-10 no modo épico). A segurança e a integridade de dados são implementadas no nível do banco, não apenas no frontend.

## Notas Técnicas

**DDL das tabelas (`supabase/migrations/001_initial_schema.sql`):**

```sql
-- Habilitar extensão para UUID
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- Tabela de jogadores
CREATE TABLE IF NOT EXISTS players (
  id          UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id     UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  name        TEXT NOT NULL CHECK (char_length(name) BETWEEN 1 AND 50),
  color       TEXT NOT NULL CHECK (color ~ '^#[0-9A-Fa-f]{6}$'),
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Tabela de partidas
CREATE TABLE IF NOT EXISTS games (
  id          UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id     UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  status      TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'finished')),
  epic_mode   BOOLEAN NOT NULL DEFAULT FALSE,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  finished_at TIMESTAMPTZ
);

-- Relação jogador-partida (nível atual)
CREATE TABLE IF NOT EXISTS game_players (
  id           UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  game_id      UUID NOT NULL REFERENCES games(id) ON DELETE CASCADE,
  player_id    UUID NOT NULL REFERENCES players(id) ON DELETE CASCADE,
  level        INTEGER NOT NULL DEFAULT 1,
  joined_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE (game_id, player_id)
);

-- Histórico de níveis (snapshots a cada 15min)
CREATE TABLE IF NOT EXISTS level_snapshots (
  id             UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  game_player_id UUID NOT NULL REFERENCES game_players(id) ON DELETE CASCADE,
  level          INTEGER NOT NULL,
  captured_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

**RLS Policies (exemplo para `players`):**
```sql
ALTER TABLE players ENABLE ROW LEVEL SECURITY;

CREATE POLICY "users can manage own players"
  ON players FOR ALL
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);
```

**Trigger de validação de nível:**
```sql
CREATE OR REPLACE FUNCTION check_level_limit()
RETURNS TRIGGER AS $$
DECLARE
  is_epic BOOLEAN;
  max_level INTEGER;
BEGIN
  SELECT epic_mode INTO is_epic FROM games WHERE id = NEW.game_id;
  max_level := CASE WHEN is_epic THEN 10 ELSE 20 END;

  IF NEW.level < 1 OR NEW.level > max_level THEN
    RAISE EXCEPTION 'Level % is out of valid range (1..%) for this game mode',
      NEW.level, max_level;
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_check_level_limit
  BEFORE INSERT OR UPDATE ON game_players
  FOR EACH ROW EXECUTE FUNCTION check_level_limit();
```
