**Status:** Ready for Review

## Contexto

Após listar jogadores (Story 2.1), o usuário precisa conseguir cadastrar novos. Esta story implementa o formulário de cadastro com nome (obrigatório, 1-30 chars) e seleção de cor identificadora. O jogador criado aparece imediatamente na lista via otimismo de UI (inserção local antes da confirmação do Supabase), garantindo feedback instantâneo.

## Notas Técnicas

**`packages/shared/src/services/players.ts` — adicionar:**
```typescript
export async function createPlayer(
  supabase: SupabaseClient<Database>,
  name: string,
  color: string
): Promise<Player> {
  const { data, error } = await supabase
    .from('players')
    .insert({ name: name.trim(), color })
    .select()
    .single();
  if (error) throw new Error(`Failed to create player: ${error.message}`);
  if (!data) throw new Error('Player creation returned no data');
  return data;
}
```

**Optimistic UI em PlayersPage:**
```typescript
const handleCreate = async (name: string, color: string) => {
  const optimisticId = `optimistic-${Date.now()}`;
  const optimisticPlayer: Player = {
    id: optimisticId, owner_id: '', name, color,
    avatar_url: null, created_at: new Date().toISOString(),
  };
  setPlayers(prev => [...prev, optimisticPlayer]);
  try {
    const real = await createPlayer(supabase, name, color);
    setPlayers(prev => prev.map(p => p.id === optimisticId ? real : p));
  } catch (e) {
    setPlayers(prev => prev.filter(p => p.id !== optimisticId));
    setError(e instanceof Error ? e.message : 'Erro ao criar jogador');
  }
};
```
