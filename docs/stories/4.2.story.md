# Story 4.2 — Otimistic UI e persistência de nível

**Epic:** EP-04 — Tela de Partida — Core
**Story ID:** EP04-S02
**Status:** Draft
**Points:** 3
**Branch:** `story/4.2-level-update`

## Contexto

O tap em +/- deve atualizar o nível em < 100ms (NFR crítico). Isso exige que a atualização local no Zustand store aconteça ANTES da chamada ao Supabase. Se a chamada ao Supabase falhar, o valor anterior é restaurado (rollback). Esta story implementa o serviço `updateLevel()` e integra o padrão optimistic UI no store.

## Notas Técnicas

**`packages/shared/src/services/levels.ts`:**
```typescript
export async function updateLevel(
  supabase: SupabaseClient<Database>,
  gamePlayerId: string,
  newLevel: number,
): Promise<void> {
  const { error } = await supabase
    .from('game_players')
    .update({ level: newLevel })
    .eq('id', gamePlayerId);
  if (error) throw new Error(`Failed to update level: ${error.message}`);
}
```

**`apps/web/src/hooks/useLevelUpdate.ts`:**
```typescript
export function useLevelUpdate() {
  const { activeGame, updatePlayerLevel } = useGameStore();

  return (gamePlayerId: string, currentLevel: number, delta: 1 | -1) => {
    const maxLevel = activeGame?.max_level ?? 10;
    const newLevel = currentLevel + delta;
    if (newLevel < 1 || newLevel > maxLevel) return;

    // Optimistic update — instantâneo
    updatePlayerLevel(gamePlayerId, newLevel);

    // Persist async — rollback se falhar
    updateLevel(supabase, gamePlayerId, newLevel).catch(() => {
      updatePlayerLevel(gamePlayerId, currentLevel); // rollback
    });
  };
}
```
