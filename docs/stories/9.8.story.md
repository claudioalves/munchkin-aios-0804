# Story 9.8 — Fix: modal de idioma exibido apenas no primeiro login

**Epic:** EP-09 — Melhorias de Experiência de Jogo
**Story ID:** EP09-S08
**Status:** Done
**Points:** 2
**Branch:** `master`

## Contexto

O modal de seleção de idioma era exibido repetidamente ao voltar ao menu, encerrar jogo ou em outras navegações. O root cause era duplo:

1. `isLanguageSet` em `LanguageContext` era uma variável derivada inline (`localStorage.getItem(...) === '1'`), não um estado React. Isso causava comportamento imprevisível como dep de `useEffect` porque o valor era recomputado a cada render sem garantia de estabilidade.

2. `isLanguageSet` estava no array de dependências do `useEffect` em `Home.tsx`. Quando o idioma era definido, `isLanguageSet` mudava de `false` para `true`, re-executando o effect. Em certas condições de timing (async `getSessionUser`), a closure capturava o valor stale.

## Root Cause Analysis

```
LanguageProvider render → isLanguageSet = localStorage.getItem() (inline)
→ isLanguageSet muda (false→true) após setLang()
→ Home re-renders (context consumer)
→ useEffect deps [isLanguageSet] detecta mudança
→ checkSession() re-executa
→ Em timing adverso: closure captura isLanguageSet=false → modal aparece novamente
```

## Fix Aplicado

1. **LanguageContext**: `isLanguageSet` convertido para `useState` com initializer:
   ```typescript
   const [isLanguageSet, setIsLanguageSet] = useState<boolean>(
     () => localStorage.getItem(LANGUAGE_SETUP_KEY) === '1'
   );
   ```
   Agora é lido do localStorage UMA VEZ (na montagem do Provider) e nunca volta a `false`.

2. **Home.tsx**: `isLanguageSet` removido das deps do `useEffect`. O effect roda apenas uma vez por mount de `Home`, usando o valor capturado na renderização inicial — que é correto, pois a pergunta "devo mostrar o modal?" deve ser respondida só na entrada da tela, não em re-renders.

## Acceptance Criteria

- [x] AC1: Modal de idioma aparece apenas no PRIMEIRO login do usuário (localStorage sem `munchkin-language-set`)
- [x] AC2: Após confirmar o idioma, nunca mais aparece (mesmo voltando ao menu, encerrando jogo, etc.)
- [x] AC3: Alterar idioma via Configurações funciona normalmente
- [x] AC4: TypeCheck e lint passam
