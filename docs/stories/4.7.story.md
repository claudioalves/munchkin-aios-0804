# Story 4.7 — Animações de entrada (stagger)

**Epic:** EP-04 — Tela de Partida — Core
**Story ID:** EP04-S07
**Status:** Draft
**Points:** 1
**Branch:** `story/4.7-animations`

## Contexto

Ao carregar a tela de partida, os PlayerCards entram em sequência (stagger) com animação `card-enter` (já definida no globals.css). O delay aumenta 60ms por card: card 1 = 0ms, card 2 = 60ms, card 3 = 120ms, etc.

## Acceptance Criteria

- [ ] AC1: Cada PlayerCard recebe `animation-delay` proporcional ao seu índice (60ms × index)
- [ ] AC2: Animação usa `animate-card-enter` já definida no globals.css
- [ ] AC3: Animação só dispara uma vez ao montar o componente (não em updates de nível)
- [ ] AC4: `npm run lint` e `npm run typecheck` passam sem erros

## Tasks de Implementação

- [ ] T1: Adicionar `style={{ animationDelay: \`${index * 60}ms\` }}` em cada card no `PlayerGrid`
- [ ] T2: Aplicar classe `animate-card-enter` ao wrapper do PlayerCard no grid
- [ ] T3: Executar `npm run lint` e `npm run typecheck`

## Notas Técnicas

No `PlayerGrid`, ao mapear os players:
```tsx
{gamePlayers.map((gp, index) => (
  <div
    key={gp.id}
    className="animate-card-enter"
    style={{ animationDelay: `${index * 60}ms` }}
  >
    <PlayerCard ... />
  </div>
))}
```
