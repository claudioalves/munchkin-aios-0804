# Story 2.4 — Cadastro Rápido Inline (New Game)

**Epic:** EP-02 — Gerenciamento de Jogadores
**Story ID:** EP02-S04
**Status:** Ready for Review
**Points:** 2
**Branch:** `story/2.4-quick-add-player`

## Contexto

Na tela de seleção de jogadores do fluxo "Novo Jogo", o usuário pode precisar adicionar um jogador de última hora sem sair do fluxo. Esta story implementa o componente `QuickAddPlayer` — um campo inline com nome + cor que cria o jogador no Supabase e o adiciona automaticamente à seleção ativa do novo jogo.

Esta story também cria a estrutura base da rota `/new-game` (passo 1 — seleção de jogadores) sem implementar o fluxo completo de novo jogo (que pertence ao EP-03). O objetivo aqui é apenas ter o contexto onde `QuickAddPlayer` se encaixa.
