# Story 1.6 — CI/CD pipeline

**Epic:** EP-01 — Setup & Infraestrutura
**Story ID:** EP01-S06
**Status:** Draft
**Points:** 2
**Branch:** `story/1.6-cicd-pipeline`

## Contexto

Nenhuma feature deve chegar à `main` sem passar por lint, typecheck e testes automatizados. Esta story configura o pipeline de CI com GitHub Actions que executa as verificações de qualidade em todo push e pull request, e o pipeline de CD que faz deploy automático na Vercel quando um merge ocorre na branch `main`.

O CI protege o repositório contra regressões e garante que os padrões definidos no coding-standards (TypeScript strict, ESLint sem erros, testes passando) sejam enforçados de forma automática — não dependendo da disciplina manual de cada desenvolvedor. O CD elimina a fricção do deploy manual, permitindo que qualquer merge na main resulte em uma versão publicada em minutos.

## Acceptance Criteria

- [ ] AC1: `.github/workflows/ci.yml` existe e é acionado em `push` e `pull_request` para qualquer branch
- [ ] AC2: O job CI executa em sequência: `npm install` → `npm run lint` → `npm run typecheck` → `npm test`
- [ ] AC3: Um push com erro de lint ou typecheck falha o CI e bloqueia o merge do PR (status check obrigatório)
- [ ] AC4: `vercel.json` existe na raiz com configuração de deploy apontando para `apps/web` como root
- [ ] AC5: O projeto está conectado ao Vercel (via `vercel link` ou importação no dashboard) com deploy automático configurado para a branch `main`
- [ ] AC6: Um merge na branch `main` aciona o deploy Vercel e a URL de preview fica disponível no PR como comentário (via Vercel GitHub integration)
- [ ] AC7: As variáveis de ambiente (`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`) estão configuradas no Vercel Dashboard — não commitadas no repositório
- [ ] AC8: O CI usa cache de `node_modules` para acelerar execuções repetidas (actions/cache para npm)

## Notas Técnicas

**`.github/workflows/ci.yml`:**
```yaml
name: CI

on:
  push:
    branches: ['**']
  pull_request:
    branches: ['**']

jobs:
  quality-check:
    name: Lint, Typecheck & Tests
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Lint
        run: npm run lint

      - name: Type check
        run: npm run typecheck

      - name: Run tests
        run: npm test
```

**`vercel.json` (monorepo com apps/web como root):**
```json
{
  "buildCommand": "npm run build --workspace=apps/web",
  "outputDirectory": "apps/web/dist",
  "installCommand": "npm ci",
  "framework": "vite",
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

**Por que `npm ci` e não `npm install` no CI:**
`npm ci` é determinístico — instala exatamente as versões do `package-lock.json` sem verificar se há atualizações. É mais rápido e seguro para CI.

**Cache de node_modules:** O `cache: 'npm'` no `actions/setup-node@v4` usa o `package-lock.json` como chave de cache. Se as dependências não mudaram, o step `npm ci` pula o download da internet e usa o cache — reduz o tempo de CI de ~2min para ~30s.

**Variáveis de ambiente no Vercel vs GitHub Actions:**
- **Vercel:** Configura `VITE_SUPABASE_URL` e `VITE_SUPABASE_ANON_KEY` em Settings → Environment Variables. Ficam disponíveis durante o build.
- **GitHub Actions:** Para uso em testes de integração futuros, adicionar como Secrets em Settings → Secrets and variables → Actions. No `ci.yml`, referenciar com `${{ secrets.SUPABASE_URL }}`.
- **Nunca** commitar `.env.local` ou valores de chave no repositório.

**Rewrite para SPA:** O `rewrites` no `vercel.json` garante que rotas do React Router (`/new-game`, `/game`) funcionem em produção sem retornar 404 — todas as rotas são redirecionadas para `index.html` e o React Router assume o controle.

**Referências:** `docs/framework/tech-stack.md` (GitHub Actions, Vercel), GitHub Actions docs: `https://docs.github.com/en/actions`, Vercel docs: `https://vercel.com/docs/monorepos`
