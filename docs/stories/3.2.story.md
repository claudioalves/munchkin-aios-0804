# Story 3.2 — Stepper Step 1 — Seleção de Jogadores

**Epic:** EP-03 — Fluxo de Novo Jogo
**Story ID:** EP03-S02
**Status:** Ready for Review
**Points:** 3
**Branch:** `story/3.2-stepper-step1`

## Contexto

A `NewGamePage` atual é um placeholder do EP-02. Esta story a substitui por um Step 1 real de um stepper de 2 passos: seleção de jogadores com validação (mínimo 2, máximo 6), `QuickAddPlayer` inline e botão "Próximo" que navega para o Step 2 (configuração).

O estado de seleção (IDs dos jogadores escolhidos) é passado via React state para o Step 2 — não envolve Supabase ainda.

## Acceptance Criteria

- [x] AC1: `/new-game` exibe indicador de passo "Passo 1 de 2: Jogadores"
- [x] AC2: Lista todos os jogadores cadastrados (de `getPlayers()`) com checkbox de seleção
- [x] AC3: Mínimo 2, máximo 6 jogadores selecionáveis — feedback visual quando limite atingido
- [x] AC4: `QuickAddPlayer` inline disponível para cadastrar jogador e já deixá-lo selecionado
- [x] AC5: Botão "Próximo →" habilitado somente quando ≥ 2 jogadores selecionados
- [x] AC6: Ao clicar "Próximo →", navega para `/new-game/config` passando `selectedIds` via state do router
- [x] AC7: Loading state enquanto busca jogadores, error state com retry se falhar
- [x] AC8: `npm run lint` e `npm run typecheck` passam sem erros
