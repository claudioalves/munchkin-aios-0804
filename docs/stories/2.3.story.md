# Story 2.3 — Remoção de Jogador

**Epic:** EP-02 — Gerenciamento de Jogadores
**Story ID:** EP02-S03
**Status:** Ready for Review
**Points:** 2
**Branch:** `story/2.3-delete-player`

## Contexto

Jogadores criados por engano ou que não participam mais precisam ser removíveis. Esta story adiciona um botão de remoção a cada `PlayerListItem` com um diálogo de confirmação simples (evita exclusão acidental) e implementa a lógica de remoção otimista.

A remoção no banco usa `ON DELETE CASCADE` — `game_players` e `level_snapshots` relacionados são excluídos automaticamente. Isso está garantido pelo schema da Story 1.3, não requer lógica adicional aqui.

## Notas Técnicas

**`packages/shared/src/services/players.ts` — adicionar:**
```typescript
export async function deletePlayer(
  supabase: SupabaseClient<Database>,
  id: string
): Promise<void> {
  const { error } = await supabase.from('players').delete().eq('id', id);
  if (error) throw new Error(`Failed to delete player: ${error.message}`);
}
```

**Optimistic removal em PlayersPage:**
```typescript
const handleDelete = async (id: string) => {
  const backup = players.find(p => p.id === id);
  setPlayers(prev => prev.filter(p => p.id !== id));
  try {
    await deletePlayer(supabase, id);
  } catch (e) {
    if (backup) setPlayers(prev => [...prev, backup].sort(/* created_at order */));
    setError(e instanceof Error ? e.message : 'Erro ao remover jogador');
  }
};
```
