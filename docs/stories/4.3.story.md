# Story 4.3 — PlayerGrid responsivo

**Epic:** EP-04 — Tela de Partida — Core
**Story ID:** EP04-S03
**Status:** Draft
**Points:** 3
**Branch:** `story/4.3-player-grid`

## Contexto

O PlayerGrid renderiza os PlayerCards em um grid CSS responsivo: 1 coluna em mobile pequeno, 2 colunas em mobile padrão, 3+ em tablet/desktop. Usa `auto-fill` com `minmax(min(100%, 280px), 1fr)` — sem breakpoints fixos, o grid se adapta naturalmente ao espaço disponível.

## Acceptance Criteria

- [ ] AC1: `PlayerGrid` existe em `apps/web/src/components/PlayerGrid/PlayerGrid.tsx`
- [ ] AC2: Grid responsivo com `grid-template-columns: repeat(auto-fill, minmax(min(100%, 280px), 1fr))`
- [ ] AC3: Cada PlayerCard recebe as props corretas: nível atual, isLeader, isVictory, callbacks
- [ ] AC4: `isLeader` calculado: jogador(es) com o maior nível entre todos (pode haver empate)
- [ ] AC5: `isVictory` calculado: nível >= `activeGame.victory_level`
- [ ] AC6: GamePage atualizada para exibir o PlayerGrid em vez do placeholder
- [ ] AC7: `npm run lint` e `npm run typecheck` passam sem erros

```typescript
interface PlayerGridProps {
  gamePlayers: GamePlayerWithInfo[];

  victoryLevel: number;

```

**Cálculo de líder:**
```typescript
const maxLevelInGame = Math.max(...gamePlayers.map(p => p.level));
const isLeader = (level: number) => level === maxLevelInGame && maxLevelInGame > 1;
```

