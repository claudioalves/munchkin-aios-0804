# Story 9.1 — Wake Lock: impedir bloqueio de tela durante o jogo

**Epic:** EP-09 — Melhorias de Experiência de Jogo
**Story ID:** EP09-S01
**Status:** InProgress
**Points:** 2
**Branch:** `story/9.1-wake-lock`

## Contexto

Durante o jogo, o smartphone bloqueia a tela por inatividade enquanto o usuário apenas observa os níveis. O Wake Lock API (`navigator.wakeLock.request('screen')`) impede isso sem necessidade de permissão explícita (apenas requer HTTPS). O lock é liberado automaticamente ao sair da tela de jogo ou quando o usuário bloqueia manualmente o dispositivo.

## Acceptance Criteria

- [x] AC2: Wake lock ativado ao montar `GamePage` e liberado ao desmontar
- [x] AC3: Wake lock re-adquirido automaticamente ao app voltar ao primeiro plano (evento `visibilitychange`)

- [x] AC5: Bloqueio manual do usuário (power button) ainda funciona normalmente — o API não interfere com isso

## Notas Técnicas

- `navigator.wakeLock` é undefined em contextos não-HTTPS (localhost funciona)
- O `WakeLockSentinel` é liberado automaticamente quando a aba perde foco; por isso o handler de `visibilitychange` re-adquire o lock
- TypeScript: `'wakeLock' in navigator` como type guard
