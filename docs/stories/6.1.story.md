# Story 6.1 — Hook useTTS

**Epic:** EP-06 — Narrador de Voz
**Story ID:** EP06-S01
**Status:** InReview
**Points:** 3
**Branch:** `story/6.1-use-tts`

## Contexto

Encapsula a Web Speech API (`window.speechSynthesis`) em um hook reutilizável. Centraliza toda a lógica de síntese de voz para que componentes como `NarratorButton` e `VictoryModal` possam consumir sem acessar a API diretamente.

## Acceptance Criteria

- [x] AC1: Hook existe em `apps/web/src/hooks/useTTS.ts` e exporta a interface descrita
- [x] AC2: `isSupported` retorna `false` graciosamente quando `window.speechSynthesis` não está disponível; demais retornos são no-ops seguros
- [x] AC3: `speak(text)` cancela qualquer fala em andamento antes de iniciar a nova
- [x] AC4: `voices` é carregado via `speechSynthesis.onvoiceschanged` e atualizado reativamente
- [x] AC5: `selectedVoiceURI` persiste no `localStorage` sob a chave `'munchkin-tts-voice'`; `setVoice(uri)` atualiza estado e localStorage
- [x] AC6: `npm run lint` e `npm run typecheck` passam sem erros

## Tasks de Implementação

- [x] T1: Criar `apps/web/src/hooks/useTTS.ts` com interface completa: `speak(text)`, `stop()`, `voices[]`, `selectedVoiceURI`, `setVoice()`, `isSupported`, `isSpeaking`
- [x] T2: Executar `npm run lint` e `npm run typecheck`

## Notas Técnicas

**Detecção de suporte:**
```typescript
const isSupported = typeof window !== 'undefined' && 'speechSynthesis' in window;
```

**Chave localStorage:** `'munchkin-tts-voice'`

**Inicialização de voices:** registrar `speechSynthesis.onvoiceschanged` para popular o array após o navegador carregar as vozes (ocorre de forma assíncrona).

**speak():** chamar `speechSynthesis.cancel()` antes de `speechSynthesis.speak(utterance)`. Selecionar a voz pelo `selectedVoiceURI` se disponível.

**isSpeaking:** derivado de `speechSynthesis.speaking`.
