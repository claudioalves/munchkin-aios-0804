**Status:** Ready for Review

**Branch:** `story/1.5-design-system`

## Contexto

O Munchkin Level Tracker tem uma identidade visual temática: tipografia medieval com Cinzel Decorative, paleta de cores de pergaminho e ouro (tons âmbar/dourados como `#c9943a`) e efeitos sutis de sombra que evocam o universo do jogo de cartas. Esta story configura o Tailwind CSS 4 com os tokens visuais definidos no Front-End Spec e cria a base CSS que todos os componentes das stories futuras irão consumir.

Uma página de teste (ou a página raiz do app) deve demonstrar visualmente que os tokens estão aplicados corretamente — cores, tipografia, espaçamentos. Isso serve como documentação viva do design system e como smoke test para garantir que `npm run dev` reflete fielmente os tokens configurados.

## Notas Técnicas

**Tailwind 4 — tokens via `@theme {}`:**
A versão 4 do Tailwind não usa `tailwind.config.ts` para tokens — usa diretivas `@theme {}` no CSS. Tokens definidos como `--color-brand-gold: #c9943a` geram automaticamente utilitários `text-brand-gold`, `bg-brand-gold`, etc.

**Google Fonts no `index.html`:**
```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Cinzel+Decorative:wght@400;700;900&family=Cinzel:wght@400;600;700&family=Crimson+Pro:ital,wght@0,400;0,500;0,600;1,400&display=swap" rel="stylesheet">
```

**Tailwind 4 com Vite:** A versão 4 do Tailwind usa plugin nativo do Vite (`@tailwindcss/vite`) em vez de PostCSS. A configuração é feita em `vite.config.ts`, não em `postcss.config.js`.
