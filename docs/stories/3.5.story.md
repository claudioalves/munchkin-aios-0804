# Story 3.5 — Continuar Partida

**Epic:** EP-03 — Fluxo de Novo Jogo

**Status:** Ready for Review

**Branch:** `story/3.5-continue-game`

## Contexto

Se o usuário fechar o app com uma partida em andamento, ao reabrir deve poder continuar de onde parou. O botão "Continuar Partida" no Menu Principal só aparece quando `getActiveGame()` retorna dados. Ao clicar, o app restaura o estado completo (game + gamePlayers) no Zustand store e navega para `/game`.

O estado persiste via dois mecanismos: Zustand localStorage (imediato) + Supabase (fonte de verdade). Esta story usa o Supabase como fonte autoritativa para garantir consistência.

## Notas Técnicas

**GamePage — lógica de restauração:**
```typescript
const { activeGame, setActiveGame, setGamePlayers } = useGameStore();
const navigate = useNavigate();

useEffect(() => {
  if (activeGame) return; // já tem no store
  getActiveGame(supabase)
    .then((game) => {
      if (!game) { navigate('/'); return; }
      setActiveGame(game);
      setGamePlayers(game.game_players);
    })
    .catch(() => navigate('/'));
}, []);
```

**Fluxo de Continuar Partida:**
1. Home: detecta `activeGame !== null` → exibe botão
2. Usuário clica → `navigate('/game')`
3. GamePage: `activeGame` já está no Zustand (localStorage) → renderiza direto
4. Se localStorage foi limpo → busca do Supabase → popula store → renderiza
