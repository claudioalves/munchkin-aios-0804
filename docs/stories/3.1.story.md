## Contexto

O Menu Principal é a tela raiz do app. A Home.tsx atual tem apenas "Novo Jogo" e "Gerenciar Jogadores" — esta story a expande para o menu completo conforme o PRD: Novo Jogo, Continuar Partida (condicional), Gerenciar Jogadores, Regras (link externo), Configurações (placeholder) e espaço para AdSense.

O botão "Continuar Partida" só aparece quando há um jogo com status='active' no Supabase. Ao carregar a Home, o app consulta `getActiveGame()` e, se retornar dados, exibe o botão.

## Notas Técnicas

**`packages/shared/src/services/games.ts`:**
```typescript
import type { SupabaseClient } from '@supabase/supabase-js';
import type { Database } from '../database.types';
import type { Game, GamePlayerWithInfo, GameWithPlayers } from '../types';

export async function getActiveGame(supabase: SupabaseClient<Database>): Promise<GameWithPlayers | null> {
  const { data, error } = await supabase
    .from('games')
    .select(`*, game_players (*, player:players (*))`)
    .eq('status', 'active')
    .maybeSingle();
  if (error) throw new Error(`Failed to fetch active game: ${error.message}`);
  return data as GameWithPlayers | null;
}
```

**Home.tsx — lógica condicional:**
```typescript
const [activeGame, setActiveGame] = useState<GameWithPlayers | null | undefined>(undefined);

useEffect(() => {
  getActiveGame(supabase)
    .then(setActiveGame)
    .catch(() => setActiveGame(null));
}, []);
```
`undefined` = loading (ainda não respondeu), `null` = sem jogo ativo, `object` = tem jogo.
