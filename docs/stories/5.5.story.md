# Story 5.5 — Limpeza de dados ao finalizar

**Epic:** EP-05 — Persistência & Salvamento
**Story ID:** EP05-S05
**Status:** Draft
**Points:** 2
**Branch:** `story/5.5-cleanup`

## Contexto

Ao finalizar a partida (HoldButton 3s ou VictoryModal), o app deve: deletar o game no Supabase (ON DELETE CASCADE cuida de game_players e level_snapshots), limpar o Zustand store (`clearGame()`), e limpar a sync queue do localStorage. Esta story verifica que o fluxo completo de limpeza está funcionando e adiciona a limpeza da sync queue.

## Acceptance Criteria

- [ ] AC1: Ao chamar `finishGame()` + `clearGame()`, o `activeGame` e `gamePlayers` são removidos do store
- [ ] AC2: A sync queue em localStorage é limpa junto com o `clearGame()` (sem writes pendentes de uma partida encerrada)
- [ ] AC3: Após finalizar, navegar para `/` e o botão "Continuar Partida" não aparece
- [ ] AC4: `clearGame()` no store inclui limpeza de `lastSavedAt` (volta para `null`)
- [ ] AC5: `npm run lint` e `npm run typecheck` passam sem erros

## Tasks de Implementação

- [ ] T1: Atualizar `clearGame()` no store para incluir `lastSavedAt: null`
- [ ] T2: Adicionar limpeza da sync queue (`localStorage.removeItem(STORAGE_KEYS.SYNC_QUEUE)`) na chamada `handleFinish` do GamePage
- [ ] T3: Verificar que VictoryModal e HoldButton ambos chamam a sequência completa de limpeza
- [ ] T4: Executar `npm run lint` e `npm run typecheck`

## Notas Técnicas

**`clearGame` atualizado:**
```typescript
clearGame: () => set({
  activeGame: null,
  gamePlayers: [],
  sortMode: 'level-desc',
  lastSavedAt: null,
}),
```

**`handleFinish` em GamePage (com sync queue cleanup):**
```typescript
const handleFinish = async () => {
  await finishGame(supabase, activeGame.id);
  localStorage.removeItem(STORAGE_KEYS.SYNC_QUEUE);
  clearGame();
  navigate('/');
};
```
