# Story 8.3 — Auto-sort por nível

**Epic:** EP-08 — Ordenação & Drag-and-Drop
**Story ID:** EP08-S03
**Status:** InReview
**Points:** 3
**Branch:** `story/8.3-auto-sort`

## Contexto

Quando `sortMode === 'level-desc'`, os cards do `PlayerGrid` sempre refletem o ranking atual (maior nível primeiro). Quando `sortMode === 'random'`, a ordem é embaralhada uma única vez ao selecionar esse modo e mantida até nova mudança.

## Acceptance Criteria

- [x] AC1: Quando `sortMode === 'level-desc'`, `gamePlayers` são exibidos ordenados por `level` decrescente; empates mantêm ordem estável
- [x] AC2: Reordenação acontece em menos de 300ms após qualquer mudança de nível (React re-render imediato)
- [x] AC3: Quando `sortMode === 'random'`, a ordem é embaralhada uma única vez ao selecionar o modo e mantida até nova seleção de modo; mudanças de nível não re-embaralham
- [x] AC4: `npm run lint` e `npm run typecheck` passam sem erros

## Tasks de Implementação

- [x] T1: Adicionar lógica de ordenação derivada no `GamePage`: `level-desc` ordena por level desc, `custom` usa ordem do store, `random` usa `randomOrder` state
- [x] T2: Embaralhamento (`random`) disparado por `useEffect` no `GamePage` ao detectar mudança de `sortMode` para `'random'`; armazena ordem embaralhada no estado local `randomOrder`
- [x] T3: Executar `npm run lint` e `npm run typecheck`

## Notas Técnicas

Fisher-Yates shuffle em `shuffleIds()`. `randomOrder` como `string[]` em estado local do `GamePage`.

## Dependências

Story 8.1 (SortDropdown / sortMode). Zustand store com `gamePlayers`.
